HTML A01 A Tríade da Web e Estrutura

A web é construída sobre três pilares fundamentais, frequentemente chamados de A Tríade da Web:

  • HTML: Estrutura e semântica (Os ossos do corpo).
  • CSS: Estilo e apresentação (A pele, roupas e aparência).
  • JavaScript: Interatividade e lógica (Os músculos e o cérebro).

HTML (HyperText Markup Language) não é uma linguagem de programação, mas sim uma linguagem de marcação. Ela usa tags para classificar os elementos na tela.

Estrutura Básica do HTML5 

<!doctype html>
<html lang="pt-BR">
  <head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Título da Página</title>
  </head>
  <body>
    <!-- O conteúdo visível vai aqui -->
  </body>
</html>

💡 Dica de Produtividade

No VS Code, crie um arquivo .html, digite ! e aperte Tab ou Enter para gerar essa estrutura automaticamente!

HTML A03 Semântica e Acessibilidade

Antes do HTML5, estruturávamos o site usando dezenas de <div> genéricas. O HTML5 trouxe tags semânticas, que descrevem seu significado tanto para o navegador quanto para o desenvolvedor. 

<body>
  <header>
    <!-- Cabeçalho com logo e menu -->
    <nav><!-- Links de navegação principal --></nav>
  </header>

  <main>
    <!-- Conteúdo principal e exclusivo da página -->
    <article>
      <!-- Um conteúdo independente que faz sentido sozinho (ex: postagem de blog) -->
    </article>

    <section>
      <!-- Agrupamento temático de conteúdo -->

CSS A06 O que é CSS, Vinculações e a Cascata

CSS (Cascading Style Sheets) transforma HTML puro em designs incríveis. Ele dita cores, layouts e fontes.

Formas de Vinculação

  • Inline: Direto na tag (Ex: <h1 style="color:red;">). Evite!
  • Interno: Na tag <style> do <head>. Médio.
  • Externo: Em arquivo .css usando <link rel="stylesheet" href="style.css">. Ideal!

Sintaxe e a Cascata 

/* Seletor { Propriedade: Valor; } */
h1 {
  color: blue;
  font-size: 24px;
}

/* Cascata: A regra definida por último (mais abaixo no código) sobrescreve a anterior se tiverem o mesmo peso */
p { color: red; }
p { color: green; } /* Os parágrafos ficarão verdes */

CSS A07 Cálculo de Especificidade e Pseudo-classes

Quando duas regras entram em conflito, o CSS calcula o "peso" delas. Isso se chama Especificidade.

Seletor Exemplo Peso
Tag h1 {} 1
Classe / Pseudo-classe .botao {} ou :hover 10
ID #cabecalho {} 100
Inline Style style="color:red;" 1000
Grito de Desespero !important 10000+

Pseudo-classes e Pseudo-elementos 

/* Estado de hover (mouse em cima) */
.botao:hover { background-color: darkblue; }

/* O primeiro filho de uma lista */
li:first-child { font-weight: bold; }

/* Injetar conteúdo via CSS */
.titulo::after { content: " ★"; color: gold; }

CSS A08 Box Model e Reset Universal

Todo elemento HTML é uma caixa retangular. Essa caixa possui 4 camadas:

  • Content (Conteúdo): O texto ou imagem real.
  • Padding (Preenchimento): Espaço transparente DENTRO da borda.
  • Border (Borda): Linha que delimita o elemento.
  • Margin (Margem): Espaço transparente FORA da borda (afasta dos vizinhos).

O Problema Matemático e o Reset CSS

Por padrão, borders e paddings aumentam o tamanho da caixa. Se você define width: 100px e dá padding: 10px, a caixa fica com 120px! Para consertar isso, usamos o "Reset Universal": 

* {
  margin: 0;
  padding: 0;
  box-sizing: border-box; /* Agora padding e border cabem dentro do width! */
}

CSS A09 Flexbox 1D: Alinhamentos Infalíveis

Flexbox é o padrão moderno para alinhar elementos em uma única direção (linha ou coluna). 

.container {
  display: flex; /* Ativa o flexbox */
  flex-direction: row; /* row (padrão) ou column */
  
  /* Alinhamento no eixo principal (Eixo X em row) */
  justify-content: center; /* center, space-between, space-around, flex-end */
  
  /* Alinhamento no eixo secundário (Eixo Y em row) */
  align-items: center; /* center, stretch, flex-start, flex-end */
  
  /* Permite que itens pulem de linha se não couberem */
  flex-wrap: wrap; 
  
  /* Espaço entre os itens (Adeus margins complexos!) */
  gap: 16px; 
}

💡 Dica para Centralizar

A forma mais fácil da história para centralizar uma div perfeitamente na tela: display: flex; justify-content: center; align-items: center; height: 100vh;

CSS A10 Grid Layout 2D: Sistemas de Tabela Complexos

Enquanto o Flexbox é para 1 dimensão, o CSS Grid brilha em layouts bi-dimensionais (linhas E colunas). 

.container-grid {
  display: grid;
  
  /* Cria 3 colunas: uma flexível(1fr), uma de 200px, e outra flexível(1fr) */
  grid-template-columns: 1fr 200px 1fr;
  
  /* Cria 2 linhas de 100px */
  grid-template-rows: 100px 100px;
  
  gap: 20px;
}

/* O filho pode ditar o tamanho que ele quer ocupar no grid! */
.item-gigante {
  grid-column: span 2; /* Ocupa o espaço de 2 colunas */
}

CSS A11 Animações e Web Responsiva

Media Queries (Mobile First)

O padrão mercado é: Escreva o CSS todo pensando em telas de Celular (Mobile First) e só depois adicione Media Queries para telas maiores. 

/* Regra padrão: Mobile */
.card { width: 100%; }

/* Quando a tela tiver PELO MENOS 768px (Tablet) */
@media (min-width: 768px) {
  .card { width: 50%; }
}

/* Quando a tela tiver PELO MENOS 1024px (PC) */
@media (min-width: 1024px) {
  .card { width: 33.33%; }
}

Transitions e Transforms 

.botao {
  background-color: blue;
  /* Anima (transiciona) qualquer mudança durante 0.3 segundos suavemente */
  transition: all 0.3s ease; 
}

.botao:hover {
  background-color: darkblue;
  /* Transform usa a placa de vídeo (GPU), sendo muito mais fluido que margins/paddings */

Cap. 1 Introdução ao JavaScript

JavaScript é a linguagem de programação da web. Criada em 1995 por Brendan Eich, ela roda nativamente em todos os navegadores modernos e, com o Node.js, também no servidor. É a única linguagem que funciona em front-end E back-end sem precisar de transpilação.

Para desenvolver JS você precisará de: um editor de código (VS Code, recomendado), um navegador moderno (Chrome, Firefox, Edge) e opcionalmente o Node.js para executar JS fora do navegador.

// No navegador: F12 → Console

O console é uma interface de depuração disponível em todos os ambientes JS. Os métodos mais usados são:

Node.js é um runtime de JavaScript para o servidor. NPM é o gerenciador de pacotes que vem junto. Com eles você pode criar servidores, scripts de automação e aplicações completas.

Cap. 3 Operadores e Expressões

3.1 Operadores Aritméticos 

const a = 10, b = 3;
console.log(a + b);  // 13 – adição
console.log(a - b);  // 7  – subtração
console.log(a * b);  // 30 – multiplicação
console.log(a / b);  // 3.333... – divisão
console.log(a % b);  // 1  – resto (módulo)
console.log(a ** b); // 1000 – potência
console.log(++a);    // 11 – pré-incremento
console.log(a--);    // 11 – pós-decremento (retorna 11, depois a=10)

3.2 Operadores de Comparação 

// == compara valor (com coerção) – EVITE
console.log(5 == "5");   // true  ← coerção!
// === compara valor E tipo – USE SEMPRE
console.log(5 === "5");  // false ✓
console.log(5 !== 6);    // true
console.log(10 > 5);     // true
console.log(3 <= 3);     // true

3.3 Operadores Lógicos e Truthy/Falsy 

// Falsy values (tudo isso é false em contexto booleano):
// false, 0, "", null, undefined, NaN

// Truthy: qualquer outro valor
Boolean("") === false
Boolean("texto") === true
Boolean(0) === false
Boolean(42) === true

// && (E): retorna o primeiro falsy ou o último valor
console.log(true && "ok");   // "ok"
console.log(false && "ok");  // false
console.log(null && "ok");   // null

// || (OU): retorna o primeiro truthy ou o último valor
console.log(false || "padrão"); // "padrão"
console.log("valor" || "pd");   // "valor"

// ?? (Nullish coalescing): só substitui null/undefined
const x = null ?? "padrão";  // "padrão"
const y = 0 ?? "padrão";     // 0 (não substitui 0!)

// ! (NÃO)
console.log(!true);  // false
console.log(!false); // true
console.log(!!0);    // false (dupla negação = boolean)

⚠️ Atenção

Sempre use === em vez de ==. O operador == faz coerção de tipo e pode gerar bugs difíceis de encontrar.

3.4 Operador Ternário e Optional Chaining 

// Ternário: condição ? valor_true : valor_false
const nota = 7;
const resultado = nota >= 6 ? "Aprovado" : "Reprovado";
console.log(resultado); // "Aprovado"

// Optional Chaining: evita erro ao acessar propriedade de null
const user = null;
// Sem ?.  → TypeError: Cannot read properties of null
// Com ?.  → undefined (seguro)
console.log(user?.nome);          // undefined
console.log(user?.address?.city); // undefined

Cap. 5 Condicionais

Condicionais permitem que o programa tome decisões diferentes dependendo de uma condição.

5.1 if / else if / else 

const nota = 75;
if (nota >= 90) {
  console.log("A – Excelente");
} else if (nota >= 70) {
  console.log("B – Bom");       // ← executa aqui
} else if (nota >= 60) {
  console.log("C – Regular");
} else {
  console.log("F – Reprovado");
}

5.2 switch 

const dia = new Date().getDay(); // 0=Dom, 1=Seg...
switch (dia) {
  case 0:
    console.log("Domingo – descanso!"); break;
  case 1:
    console.log("Segunda – produtividade!"); break;
  case 5:
  case 6:
    console.log("Fim de semana!"); break;
  default:
    console.log("Dia útil – foco!");
}

💡 Dica

Sempre use break no switch, exceto quando quiser que múltiplos cases sejam tratados juntos (como sábado e domingo acima).

5.3 Guardas antecipados (Early Return) 

// Em vez de:
function processar(usuario) {
  if (usuario !== null) {
    if (usuario.ativo) {
      // lógica principal...
    }
  }
}
// Prefira: retorno antecipado (early return)
function processar(usuario) {
  if (!usuario) return;
  if (!usuario.ativo) return;
  // lógica principal sem aninhamento excessivo
  console.log("Processando:", usuario.nome);

  const users = getUsuarios();

function login(email, senha) {
  const users = getUsuarios();
  const user = users[email];
  if (!user || user.senha !== senha) throw new Error("Credenciais inválidas");
  localStorage.setItem(SESSAO_KEY, JSON.stringify({ email, nome: user.nome }));
  return user;
}
function logout() {
  localStorage.removeItem(SESSAO_KEY);
}
function getSessao() {
  return JSON.parse(localStorage.getItem(SESSAO_KEY) || "null");
}

    H1. Introdução ao HTML

    HTML (HyperText Markup Language) não é uma linguagem de programação, mas sim uma linguagem de marcação. Ela dita a estrutura e o significado (semântica) do conteúdo na web, permitindo que navegadores leiam e exibam as informações textuais, links e imagens adequadamente.

    H1.1 A Tríade da Web

    Para entender o Front-end moderno, você deve pensar em três camadas fundamentais trabalhando juntas:

    • HTML (Os ossos): Define "O QUE" está na tela. O texto, a imagem, o botão.
    • CSS (A pele/roupa): Define "COMO" as coisas aparecem. As cores, o layout, a responsividade.
    • JavaScript (O músculo): Define "O QUE FAZ". Interatividade, requisições de dados, lógica.

    C1. Introdução e Cascata

    C1.1 Como o CSS Funciona

    CSS (Cascading Style Sheets) dita a aparência do HTML. O "Cascading" (Cascata) significa que as regras aplicadas de cima para baixo importam, e estilos em elementos "pais" fluem para os "filhos", a menos que o filho tenha uma regra contrária. 

    /* Sintaxe básica CSS: SELETOR { PROPRIEDADE: VALOR; } */
body {
  background-color: #f4f4f4;
  font-family: Arial, sans-serif; /* A cascata leva a font para todos os filhos de 'body'! */
  color: #333;
}

    💡 Formas de adicionar CSS

    1) Inline: style="..." direto na tag HTML. 2) Interno: Dentro da tag <style> no head do arquivo. 3) Externo: via arquivo separado <link rel="stylesheet" href="style.css"> (O único recomendado para projetos reais!).

    C2. Seletores e Especificidade

    C2.1 Hierarquia de Cores (Especificidade)

    Se você manda um texto ficar azul por estar numa tag genérica, mas mandar também ficar vermelho por ele ter uma "classe", quem ganha a briga? A Especificidade. O ID ganha das Classes que ganham da Tag pura, no modelo de pontuação [0, 0, 0]. 

    <!-- O elemento -->

/* 1. Tag (Especificidade Mínima: 1 ponto) */

.btn-ativo { color: blue; }

/* 3. ID (Especificidade Alta: 100 pontos) - Vence de ambas acima! O texto será CINZA */
#cabecalho { color: gray; }

/* !important: Vence todos, ignora as regras (Só use se for extremamente necessário) */
p { color: red !important; }

    C3. Box Model e Layout

    C3.1 Nós vivemos num quadrado web

    Nunca se esqueça: Tudo, absolutamente tudo no desenvolvimento visual web é renderizado como CAIXAS (Quadrados/Retângulos). Uma fonte arredondada? Está num retângulo transparente. Um botão perfeitamente circular? É um quadrado CSS que te engana com bordas 100% arredondadas. O Box Model compreende 4 áreas:

    • Content (Conteúdo puro): O tamanho exato do texto ou da imagem em si.
    • Padding (Enchimento interno): O "ar" dentro da caixa protegendo o conteúdo encostar nos muros (borda). O botão é gordinho por causa disso.
    • Border (Muro): A barreira divisora visível (opcional).
    • Margin (Espaço externo alheio): O espaço FORA da caixa usado para repelir (afastar) outras caixas web vizinhas.

    C3.2 O Reinício Universal CSS Insubstituível 

    /* ESTE CÓDIGO VAI NO TOPO DE QUALQUER PROJETO QUE VOCÊ FOR FAZER NA VIDA */
* {
  box-sizing: border-box; /* Garante que 'padding' não arrebente a largura total que você declarou na caixa. A matemática faz sentido do lado de dentro. */
  margin: 0;  /* Remove os espaços que os navegadores (Chrome/Safari) inventam nos parágrafos por conta própria */
  padding: 0;
}

.caixa-linda {
  width: 100%;       /* Ocupe tudo! */
  max-width: 500px;  /* Vá até no máximo 500 pixels nas telas grandes */
  padding: 24px;   
  border: 1px solid #ddd; 
  margin: 16px auto; /* 16px empurrando vizinhos ACIMA/ABAIXO, mas Centralize a mim (auto nas LATERAIS)! */
  border-radius: 8px; /* Arredonde os cantos! */
}

    ⚠️ Margin Collapse

    Cuidado: Quando você tem uma caixa empurrando margin-bottom: 20px pra baixo, e uma caixa vizinha logo em seguida empurrando margin-top: 20px pra cima... Elas não somam 40px! O navegador vai pegar exclusivamente o maior e sobrepor. O espaço será de apenas 20px (Elas "colapsam/fundem").

    C4. Flexbox (Caixa Flexíveis)

    C4.1 Alinhamento Perfeito em 1 Dimensão

    O Flexbox resolveu o maior problema do Front-end moderno: alinhar coisas facilmente sem gambiarras. Ele constrói coisas em um eixo (ou tudo em Linha, ou tudo em Coluna). 

    /* O Elemento "Pai" (Container) que dita as regras do Flow */
.container {
  display: flex; /* MÁGICA LIGADA AQUI */
  
  flex-direction: row;     /* Padrão (Lado a Lado). Troque por 'column' para empilhar. */
  justify-content: center; /* Alinhamento ao longo da Direção Principal */
  /* center | space-between (Espalha itens para as bordas) | space-around */
  
  align-items: center;     /* Alinhamento ao longo da Direção Cruzada */
  /* center (Centraliza tudo verticalmente, o Graal da Web) | stretch (Estica) */
  
  gap: 16px;       /* O moderno espaço cimentado entre os itens! */
  flex-wrap: wrap; /* Se os itens não couberem na linha, jogue o resto para a linha de baixo! */
}

/* Os Elementos "Filhos" (Items) obedecem o Pai e crescem sob demanda */
.item {
  flex: 1; /* Todos com 'flex: 1' dividem 100% da tela irmanamente em pedaços rigorosamente iguais */
}

    C4.2 Centralizando uma Div em 3 Linhas

    A pergunta mais feita em fóruns de web dev na era pré-Flexbox: 

    .fundo-tela-inteira {
  height: 100vh; /* 100% da Altura Visual da Janela (View Height) */
  display: flex;
  justify-content: center; /* Centro Horizontal (Eixo X na Row) */
  align-items: center;     /* Centro Vertical (Eixo Y na Row) */
}

    C5. Grid Layout (Tabelas e Seções Puras)

    C5.1 Desvendando as Dimensões

    Enquanto o Flexbox resolve fluxos de 1 dimensão (Só Linha, Ou Só Coluna), o CSS Grid é feito para 2 dimensões simultâneas. Ele permite a criação da "Tabela" (Super grade) perfeita para construir galerias de imagens, dashboards ou o esqueleto estrutural Master de um sistema. 

    .grade-fotos {
  display: grid;
  
  /* CRIA 3 COLUNAS. As duas da ponta terão 200px fixos. A do meio,
     usando a Fração Mágica (1fr) consumirá o 100% de todo o espaço que sobrar. */
  grid-template-columns: 200px 1fr 200px;
  
  /* Crie uma repetição de 4 colunas iguais da forma flexível usando FRAÇÕES */
  /* grid-template-columns: repeat(4, 1fr); */
  
  gap: 20px; /* Grade também tem um espaço inteligente sem margens dolorosas */
}

    .layout-sistema {
